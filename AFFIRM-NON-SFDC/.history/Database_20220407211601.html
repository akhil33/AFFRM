1) If you designed the database, would you have done anything differently? What and why?

I would have redesigned the data model to have 3 tables namely 1. 'Movie' 2. 'Movie Cast and Crew' 3. 'Person'

1. Movie - The Movie table will almost remains the same.
2. Movie Cast and Crew - This will be the juntion Objects to hold cast informaion for the perticular Movie. Junction is ideal in this situation since a movie can have n cast, and a person can be part of n number of movie.
3. Person - This will hold the information all individuals or the crew. This eleminated the need of assigning the perticular role of an idivual since the above juntion objects can have data point to assign role per mover per person

Movie Cast and Crew will have following field (columns):

a. Movie: Lookup to Movie
b. Crew: Lookup to Person
c. Role Type: May be unrestricted picklist Field
d. Character Name: String (text)

The new data model has following advantages

1. better maintanbility
2. True representation
3. Reduced data redundancy
